## CSV Event Logs

### Daily CSV Logs
**Location:** `/Users/sethrose/Developer/Github/scanner-feed/logs/`

**Files:** `scanner-events-YYYY-MM-DD.csv`

**Content:** Detailed event data including:
- Timestamp
- Raw transcript
- Normalized transcript
- Radio codes
- Radio numbers
- HAM callsigns
- Weather severity
- RMS (audio level) metadata
- Model used
- Segment age

**CSV Columns:**
| Column | Description |
|--------|-------------|
| timestamp | Event timestamp |
| feed_name | Audio feed name |
| raw_text | Original transcription |
| normalized_text | Normalized transcription |
| radio_codes | Detected radio codes |
| radio_numbers | Radio unit numbers |
| ham_callsigns | HAM radio callsigns |
| weather_severity | Weather alert level |
| rms_level | Audio level (0.0-1.0) |
| model | MLX model used |
| segment_age | Age of audio segment |

**Viewing CSV Logs:**
```bash
# List all CSV logs
ls -lh /Users/sethrose/Developer/Github/scanner-feed/logs/

# View today's log
ls -t /Users/sethrose/Developer/Github/scanner-feed/logs/scanner-events-*.csv | head -1

# View with column headers
head -1 $(ls -t /Users/sethrose/Developer/Github/scanner-feed/logs/scanner-events-*.csv | head -1)

# View first 10 rows
head -n 10 $(ls -t /Users/sethrose/Developer/Github/scanner-feed/logs/scanner-events-*.csv | head -1)

# Count events in today's log
wc -l $(ls -t /Users/sethrose/Developer/Github/scanner-feed/logs/scanner-events-*.csv | head -1)

# Search for specific text in CSV logs
grep -i "10-4" /Users/sethrose/Developer/Github/scanner-feed/logs/scanner-events-*.csv
```

### CSV Log Retention
The system automatically manages CSV log retention:
- **Retention period:** 14 days (configurable via `CSV_RETENTION_DAYS`)
- **Max total size:** 500MB (configurable via `CSV_MAX_TOTAL_MB`)
- **Compression:** Logs older than 2 days are gzipped (configurable via `COMPRESS_OLD_CSV_DAYS`)

## Docker Logs

### Container Logs
**Location:** Docker container stdout/stderr

**Viewing:**
```bash
# Watch logs in real-time
docker compose logs -f scanner

# View last 100 lines
docker compose logs --tail 100 scanner

# View logs since last hour
docker compose logs --since 1h scanner

# Search logs
docker compose logs scanner | grep -i "error"
```

### Log Levels
The scanner worker logs different levels of information:
- **INFO:** Normal operations, feed connections, transcription results
- **WARNING:** Audio level issues, feed connection problems
- **ERROR:** Transcription failures, system errors

```bash
# Create archive of logs older than 7 days
find /Users/sethrose/Developer/Github/scanner-feed/logs/ -name "*.csv" -mtime +7 -exec tar -czf ~/Desktop/scanner-logs-$(date +%Y%m%d).tar.gz {} +
```

## Troubleshooting Data Issues

If you're having trouble finding or accessing data:

1. **Check directory permissions:**
   ```bash
   ls -la /Users/sethrose/Developer/Github/scanner-feed/
   ```

2. **Verify Docker is running:**
   ```bash
   docker compose ps
   ```

3. **Check disk space:**
   ```bash
   df -h /Users/sethrose/Developer/Github/scanner-feed/
   ```

4. **View recent errors:**
   ```bash
   docker compose logs --tail 50 scanner | grep -i error
   ```
